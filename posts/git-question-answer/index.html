Git Questions And Answers

September 12, 2020 One-minute read

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。 分布式相比于集中式的最大区别在于开发者可以提交到本地，每个开发者通过克隆操作，在本地机器上拷贝一个完整的Git仓库。

Git 常见问题 Q&A

💡 git log 和 git reflog的区别 ？

git log 命令显示从最近到最远的提交日志，git reflog命令显示所有的历史操作，包括 git reset 命令中被删除的 commit 记录。

💡 git merge 和 git rebase 的区别 ？

git merge 合并两个分支，并生成一个新的节点，而 git rebase 则不会生成新的节点，而是将两个分支融合成一个线性的提交。

💡 git reset HEAD^ 和 git reset --hard HEAD^ 的区别 ？

都是用来撤销操作的，前者只撤销 commit 不会删除已经修改了的内容，后者撤销 commit 并且将文件内容恢复为上次 commit 时的内容。
